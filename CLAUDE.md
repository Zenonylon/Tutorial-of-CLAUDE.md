# Tutorial Project

## Rules

- Always answer in Korean.
- Explain your plan before modifying code.
- Explain which files will be changed.
- Explain expected impact of the change.

#Development Rules

## Before Editing

Before modifying any file:

- Explain why the modification is needed.
- List files that will be modified.
- Describe expected impact on performance and functionality.

## After Editing

After every code change, generate a short report in the following format:

### Change Summary
- What was changed

### Modified Files
- List of modified files

### Reason
- Why the change was made

### Expected Impact
- Performance impact
- Functional impact

### Risks
- Possible side effects

### Verification
- How to test the change

Keep reports concise and technical.
Do not skip the report.
